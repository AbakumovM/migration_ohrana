# Охрана — система управления объектами

Веб-приложение для учёта клиентов и охраняемых объектов Ревдинского ОВО.
Миграция с Visual FoxPro 8 на Python/FastAPI.

---

## Возможности

- **Юридические лица** — список с поиском по названию и номеру договора, создание, редактирование, удаление
- **Объекты охраны** — карточка объекта, виды охраны (КТС/ПЦН), тарифы, расписание, архив
- **Документы PDF** — перечень, расчёт, протокол согласования цены
- **Настройки** — реквизиты организации

## Запуск

### macOS

```bash
# Установить зависимости для WeasyPrint (генерация PDF)
brew install pango

# Установить зависимости проекта
uv sync

# Запустить
uv run python run.py
```

### Linux (Ubuntu / Debian / Astra Linux) — готовая сборка

#### Вариант 1: запуск из терминала

Скачать готовую сборку из раздела [Actions](../../actions) (артефакт `ohrana-linux`), распаковать и запустить:

```bash
tar -xzf ohrana-linux.tar.gz -C ~/ohrana --strip-components=1
~/ohrana/ohrana
```

- Браузер откроется автоматически
- Для остановки: `Ctrl+C` в терминале
- База данных хранится в `~/ohrana/ohrana.db` и не затирается при обновлении

#### Вариант 2: запуск двойным кликом (без терминала)

Этот вариант подходит если нет доступа к терминалу.

**Шаг 1.** Распаковать архив так, чтобы получилась следующая структура:

```
~/ohrana/
├── ohrana        ← бинарник (и все файлы из архива)
└── ohrana.db     ← база данных (передаётся отдельно)
```

**Шаг 2.** Создать файл `ohrana.desktop` в той же папке `~/ohrana/` со следующим содержимым (заменить `имя_пользователя` на реальное):

```ini
[Desktop Entry]
Name=Охрана
Exec=/home/имя_пользователя/ohrana/ohrana
Type=Application
Terminal=false
```

> Имя пользователя можно узнать: открыть файловый менеджер, он обычно показывает путь вида `/home/ivan/` — это и есть имя.

**Шаг 3.** Двойной клик на `ohrana.desktop` — браузер откроется автоматически.

**Остановка приложения:** закрыть браузер недостаточно — сервер продолжает работать. Чтобы завершить, нужно найти процесс `ohrana` в системном мониторе и завершить его, либо перезагрузить компьютер.

### Linux (Ubuntu / Debian) — запуск из исходников

```bash
sudo apt install -y libpango-1.0-0 libpangocairo-1.0-0 libcairo2 \
    libgdk-pixbuf2.0-0 libffi-dev shared-mime-info

uv sync
uv run python run.py
```

### Windows

Пользователям Windows не нужно ничего устанавливать — скачать готовую сборку из раздела [Actions](../../actions) (артефакт `ohrana-windows`), распаковать и запустить `ohrana.exe`.

- Браузер откроется автоматически
- Иконка приложения появится в системном трее (правый нижний угол)
- Для выхода: правый клик на иконку → **Выход**
- База данных хранится в `C:\Users\ИмяПользователя\ohrana\ohrana.db` и не затирается при обновлении

## Сборка бинарных пакетов

Сборка запускается автоматически через GitHub Actions.

**Разово (по тегу):**
```bash
git tag v1.0
git push --tags
```

**Вручную:** Actions → нужный workflow → Run workflow.

| Workflow | Артефакт | Платформа |
|---|---|---|
| Build Windows EXE | `ohrana-windows` | Windows 10/11 |
| Build Linux | `ohrana-linux` | Ubuntu 22.04 / Debian / Astra Linux |

Скачать результат: вкладка Actions → нужный запуск → Artifacts.
